# 🐭 ドロボーマウス 🐱

2チーム対戦（2人 vs 2人）の攻守交代制ゲーム。舞台は猫が運営するスーパー。
ネズミ側はNPCネズミに紛れて商品を盗んで出口から持ち出すとポイント獲得。
猫側は監視カメラ越しに怪しいネズミを見つけて「ダウト」（クリック）で暴く。

- Three.js（3D描画）+ Firebase Realtime Database（同期）+ Vite + TypeScript
- ボイスチャットはZoom等の外部サービスで代用（チームごとに別ルームを作る）

## ルール

- **1ラウンド1分 × 2ラウンド**。前半はチームAがネズミ（攻撃）・チームBが猫（守備）、後半で攻守交代
- **ネズミ**: お題の棚（光る場所）に5秒とどまると盗み成功。その間**体が左右にわずかに揺れて目立つ**。盗んだ商品を出口（緑ゲート）から持ち出すと**1ポイント**。持ち出した後は**カメラの死角にリスポーン**して何度でも狙える
- **猫**: 2人とも監視カメラ役。店内カメラ12台を**オンライン⇔オフラインでトグル**（同時オンラインは1人4台まで）。カメラ映像の怪しいネズミをクリックすると「ダウト」
  - プレイヤーのネズミを当てたら**そのラウンドは即終了**
  - NPCだったら空振り。ダウトは**1ラウンドにつき各プレイヤー2回まで**
- **監視の可視化**: オンラインのカメラは**ポール上部の球体が赤く発光**し、**視野に入っている床が明るくなる**。ネズミは「みられている場所」と死角を見分けながら動く
- **勝敗**: 2ラウンドの合計ポイントが多いチームの勝ち（同点は引き分け）

| 役割 | 操作 |
| --- | --- |
| 🐭 ネズミ | WASD / 矢印キーで移動 |
| 🎥 猫（カメラ監視） | 下部ボタン or 数字キーでカメラのオン/オフ切替（4台まで）、映像内のキャラをクリックでダウト、Mでカメラマップ（中央モーダル） |

- お題（どの商品をどの棚で盗むか）はネズミにしか見えない。盗みが成立するたびに次のお題に更新される
- ゲームバランスは `src/config.ts` の数値で調整できる

## セットアップ

```bash
npm install
npm run dev
```

### ローカルモード（Firebase不要・すぐ試せる）

`.env` が無い場合は自動でローカルモードになり、**同一PCのブラウザタブ間**で同期する。
`npm run dev` して同じURLをタブで4つ開き、同じ部屋コードで入れば1人で動作確認できる。

### オンラインモード（Firebase）

1. [Firebaseコンソール](https://console.firebase.google.com/)でプロジェクト作成
2. Realtime Database を作成（テストモードでOK。ロケーションは asia-southeast1 など）
3. プロジェクトの設定 > マイアプリ > ウェブアプリを追加して設定値を取得
4. `.env.example` を `.env` にコピーして値を記入
5. `npm run dev` → 各プレイヤーが同じ部屋コードで参加

プロトタイプ用のDBルール（全公開なので遊び終わったら閉じること）:

```json
{
  "rules": {
    "rooms": { ".read": true, ".write": true }
  }
}
```

## 構成

```
src/
├── main.ts          エントリポイント（ロビー⇔ゲーム⇔ラウンド切替の画面遷移）
├── config.ts        ゲームバランス定数（盗み時間・ラウンド時間・ダウト回数など）
├── types.ts         チーム・役割・イベント等の型定義
├── net/
│   ├── adapter.ts   同期インターフェース
│   ├── firebase.ts  Firebase Realtime Database実装
│   └── local.ts     BroadcastChannel実装（同一PCタブ間・開発用）
├── game/
│   ├── game.ts      ゲーム本体（ループ・盗み/ダウト判定・ラウンド進行・勝敗）
│   ├── world.ts     店内の3D構築（棚・壁・出口・防犯カメラ・盗みスポット）
│   ├── npc.ts       NPCネズミ（seedから決定論的に動くので通信同期が不要）
│   ├── cctv.ts      防犯カメラビュー（同時4台の2x2グリッド・接続切替）
│   └── controls.ts  キーボード入力
└── ui/
    ├── lobby.ts     ロビー（部屋参加・チーム選択・開始）
    ├── map.ts       カメラマップ（視野と死角の俯瞰図）
    └── hud.ts       ゲーム中HUD（タイマー・スコア・お題・通知）
```

## 実装メモ

- **NPCの同期**: NPCはゲームseedから決定論的に動く（`NpcSim.posAt(t)`）。全クライアントが同じ計算をするため位置の通信が不要。seedはラウンドごとに変わる
- **お題の秘匿**: お題の場所・商品名はネズミのクライアントでのみ描画される（プロトタイプなのでクライアント側秘匿）
- **スコア**: `escape` イベントの積み上げから全クライアントが同じ集計をする。イベントは試合を通して保持し、ロビーに戻るときに消す
- **ラウンド進行**: 時間切れはホスト（不在時は2秒後に誰でも）、ダウト成功は当てた猫が `phase` を書き込む楽観的な方式（プロトタイプ想定）
- **ダウト判定**: CCTVのクリック位置から該当カメラのNDC座標を求めてレイキャスト。プレイヤーのネズミに当たれば `caught`、NPCなら `miss`
- **監視状態の同期**: 各猫のオンラインカメラIDを `rooms/{room}/cams/{pid}` に書き、全クライアントが購読して球の発光と床ハイライトに反映する
- **視野と死角**: カメラの実効視野はマップ表示・床ハイライト・死角リスポーン探索で同じ2Dレイキャスト（`castRay`）を使って一貫させている。死角リスポーンは全12台から見えない歩行可能地点を起動時に自動探索する
