# 🐭 ドロボーマウス 🐱

ネズミ2匹 vs 猫2匹の4人対戦ゲーム。舞台は猫が運営するスーパー。
ネズミはNPCネズミに紛れて商品を盗み、出口から脱出したら勝ち。
猫は「カメラ監視役」と「鬼」のペアでネズミを見つけてキャッチしたら勝ち。

- Three.js（3D描画）+ Firebase Realtime Database（同期）+ Vite + TypeScript
- ボイスチャットはZoom等の外部サービスで代用（ネズミ同士・猫同士で別ルームを作る）

## 遊び方

| 役割 | 目的 | 操作 |
| --- | --- | --- |
| ネズミ 1 / 2 | お題の商品の棚（光る場所）に10秒とどまると盗み成功。何か持った状態で出口（緑ゲート）に入れば勝ち | WASD / 矢印キーで移動 |
| 猫（鬼） | NPCの中からプレイヤーのネズミを見つけてキャッチ。速度はネズミの2倍。NPCを誤キャッチすると3秒硬直 | WASD / 矢印キーで移動、スペースでキャッチ |
| 猫（カメラ監視） | 店内6台の防犯カメラを監視して鬼に指示。怪しいカメラに「アラート」を送ると鬼の画面に通知が出る | クリック / 1〜6キーでカメラ選択、Gでグリッド⇔単一表示切替 |

- お題（どの商品をどの棚で盗むか）はネズミにしか見えない。盗みが成立するたびに次のお題に更新される
- 制限時間は5分。時間切れは猫の勝ち
- ゲームバランスは `src/config.ts` の数値で調整できる

## セットアップ

```bash
npm install
npm run dev
```

### ローカルモード（Firebase不要・すぐ試せる）

`.env` が無い場合は自動でローカルモードになり、**同一PCのブラウザタブ間**で同期する。
`npm run dev` して同じURLをタブで4つ開き、同じ部屋コードで入れば1人で動作確認できる。

### オンラインモード（Firebase）

1. [Firebaseコンソール](https://console.firebase.google.com/)でプロジェクト作成
2. Realtime Database を作成（テストモードでOK。ロケーションは asia-southeast1 など）
3. プロジェクトの設定 > マイアプリ > ウェブアプリを追加して設定値を取得
4. `.env.example` を `.env` にコピーして値を記入
5. `npm run dev` → 各プレイヤーが同じ部屋コードで参加

プロトタイプ用のDBルール（全公開なので遊び終わったら閉じること）:

```json
{
  "rules": {
    "rooms": { ".read": true, ".write": true }
  }
}
```

## 構成

```
src/
├── main.ts          エントリポイント（ロビー⇔ゲームの画面遷移）
├── config.ts        ゲームバランス定数（盗み時間・速度・制限時間など）
├── types.ts         役割・イベント等の型定義
├── net/
│   ├── adapter.ts   同期インターフェース
│   ├── firebase.ts  Firebase Realtime Database実装
│   └── local.ts     BroadcastChannel実装（同一PCタブ間・開発用）
├── game/
│   ├── game.ts      ゲーム本体（ループ・盗み/キャッチ判定・勝敗）
│   ├── world.ts     店内の3D構築（棚・壁・出口・防犯カメラ・盗みスポット）
│   ├── npc.ts       NPCネズミ（seedから決定論的に動くので通信同期が不要）
│   ├── cctv.ts      防犯カメラビュー（グリッド/単一表示・アラート送信）
│   └── controls.ts  キーボード入力
└── ui/
    ├── lobby.ts     ロビー（部屋参加・役割選択・開始）
    └── hud.ts       ゲーム中HUD（タイマー・お題・進捗・通知）
```

## 実装メモ

- **NPCの同期**: NPCはゲームseedから決定論的に動く（`NpcSim.posAt(t)`）。全クライアントが同じ計算をするため位置の通信が不要
- **お題の秘匿**: お題の場所・商品名はネズミのクライアントでのみ描画される（猫には送られても表示されない。プロトタイプなのでクライアント側秘匿）
- **勝敗判定**: 検知したクライアントが `phase` を `ended` に書き込む楽観的な方式（プロトタイプ想定）
